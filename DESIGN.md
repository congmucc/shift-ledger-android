# Shift Ledger Design System / 工时账本设计基线

## 1. Visual Theme & Atmosphere

Shift Ledger 是一本私人“工时账本”。它不是企业考勤后台，也不是排班审批系统。设计目标是让用户快速确认：

- 今天有没有记完整；
- 哪天上了哪些班、几段、多久；
- 加班、夜班、补贴、扣款和收入估算是否可信；
- 数据是否能导出、备份和恢复。

整体气质改为：**iOS Neutral、白底为主、低噪音、数字可信、触控友好、长期使用不疲劳**。

本项目不再使用大面积暖纸面色作为默认背景。界面应接近现代手机工具 App：白色或极浅灰背景、白色卡片、清晰边框、少量系统感蓝色作为主操作，绿色只表达完成/加班/正向收入。颜色是状态和层级工具，不是装饰。

### 设计关键词

- Clean mobile utility / 干净的移动工具
- iOS neutral / 白灰系统感
- Trustworthy ledger / 可信账本
- Low-pressure personal tracking / 低压力私人记录
- Dynamic work segments / 可承载多段与多次加班

## 2. Information Architecture

底部导航采用五个视觉位置：

1. **首页**：今日记录、本周期进度、快捷操作。
2. **日历**：月视图/列表视图、今日跳转、日期详情、补一段。
3. **中间新增**：主操作入口，打开新增/编辑工时记录 sheet。
4. **汇总**：本月/本周/年度/发薪周期/自定义统计和 CSV 导出。
5. **设置**：班次模板、计薪规则、规则历史、加班规则、夜班规则、发薪周期、导出与备份、WebDAV。

每个页面先满足一个首要问题，再承载次要能力；实现时不得用“多放几个入口”代替清晰路径。

| 页面 | 用户第一眼要确认 | 首屏主行动 | 次级内容 | 不应抢占首屏的内容 |
| --- | --- | --- | --- | --- |
| 首页 | 今天记了几小时、有没有漏记 | 补一段 | 今天分段、本周期进度、看日历/看汇总 | 过多快捷入口、导出和设置类动作 |
| 日历 | 哪些日期有工时、当前选中哪天 | 选日期或补一段 | 月汇总、日期详情、列表视图 | 月统计卡片压过日历网格 |
| 汇总 | 这个范围总工时和收入估算是否可信 | 切换范围、查看明细 | 异常日期、备注/补贴/扣款/加班下钻、CSV 导出 | 装饰性图表、不可下钻的大数字 |
| 设置 | 规则、备份和数据出口是否可控 | 编辑当前规则或配置备份 | 规则历史、WebDAV、自动备份状态 | 与记录无关的高级配置 |
| 新增/编辑 sheet | 当前日期和分段是否正确 | 保存 | 新增分段、编辑单段、计薪提示 | 破坏性删除与保存同级并列 |

实现位置：`lib/src/ui/theme.dart`。

### Color Rules

- 白色和浅灰是默认视觉基底；不要再使用大面积暖黄、米色、复古纸面色。
- 蓝色只用于主操作和当前选中，不用于普通装饰。
- 普通工时使用蓝色系，但要比主按钮弱。
- 加班使用绿色系；多次加班不得挤进固定两格布局，应通过 chip 和分段列表表达。
- 夜班使用 indigo，不使用黑底大面积压暗页面。
- 琥珀/橙色只用于提醒，不用于主按钮。
- 红色只用于危险、失败、删除、覆盖。
- 不只靠颜色表达状态；必须同时使用文字、图标、短线、chip 标签或语义说明。
- 正文和关键数字对比度至少满足 WCAG 4.5:1；大号粗体数字至少满足 3:1。

## 4. Typography Rules

当前实现使用 `ShiftLedgerCJK` 作为中文字体，并保留系统字体 fallback。继续使用 CJK 字体，不引入花哨字体。

- 页面标题：28sp，粗字重，短标题优先，颜色 `ink`。
- Section 标题：20-22sp，粗字重，颜色 `ink`。
- 首页和汇总核心数字：36-54sp，超粗字重，使用 tabular figures。
- 卡片主值：22-30sp，粗字重，数字对齐。
- 正文：15-16sp，行高 1.45-1.55。
- 辅助标签：12-13sp，颜色 `muted`，不承载关键结果。
- 底部导航：11-12sp，粗字重，最大缩放受控，但不能隐藏语义。

数字是本产品的视觉主角。工时、收入、天数、次数必须稳定对齐；不要让数字像普通段落文字一样漂浮。

## 5. Component Stylings

### PageFrame

- 页面统一容器。
- 背景使用 `background` 或 `canvas`。
- 顶部标题清晰，右侧动作保持 44dp 触控目标。
- 底部留出导航栏安全空间。

### LedgerCard

- 默认卡片组件。
- 背景 `surfaceRaised` 或 `surface`。
- 20-24dp 圆角。
- 1dp `hairline` 边框。
- 阴影极轻或无阴影，避免旧版纸面厚重感。
- 卡片用于首页指标、日期详情、设置分组、汇总区块。

### MetricCard

- 用于核心数字。
- 首页“今日已记录”和汇总核心指标优先使用。
- 核心数字要有足够尺寸和留白。
- 可点击时使用明确的文字或箭头提示，不只靠阴影。

### Status Chips

- 用于表达普通、加班、夜班、补贴、扣款、完成、等待、失败等状态。
- 形状为 pill-shaped。
- 支持自动换行，不允许固定两列或固定两个状态。
- 单个 chip 文案短，格式优先为：`普通 7.5h`、`加班 1.0h`、`夜班 1次`、`补贴 ¥20`。

### WorkEntryTile

- 展示单段工时。
- 必须包含时间范围、类型、净工时、地点/岗位、补贴/扣款/备注等摘要。
- 左侧使用 4dp 状态短线：普通蓝、加班绿、夜班 indigo。
- 如果一天有多次加班，按真实 segment 展示，不合并成一个固定位置的格子。
- 编辑按钮触控目标不小于 44dp。

### Calendar Cell

- 使用 `table_calendar` 承载网格。
- 每个日期需要语义化描述：日期、工时、段数、是否加班/夜班/备注/今日。
- 普通日：浅蓝背景或蓝色短标记。
- 加班日：浅绿背景或绿色短标记。
- 夜班日：浅 indigo 背景或 indigo 标记。
- 今日：蓝色描边。
- 选中日：更强蓝色描边 + 白底，避免和状态色冲突。
- 备注：小点或图标，不只靠颜色。

### Bottom Navigation

- 视觉上有 5 个位置，中间“＋”是主操作，不对应普通页面。
- 背景白色或 `surfaceRaised`，边框 `hairline`，轻微浮层。
- 中间新增按钮使用 `primaryBlue`，白色加号。
- 当前 tab 使用蓝色文字或浅蓝 pill，不能只改变透明度。
- 所有入口触控区域不小于 44dp。

### Buttons

- Primary：`primaryBlue` 背景，白字，pill-shaped 或 16-20dp 圆角。
- Secondary：白底或浅灰底，`hairline` 边框，`ink` 文案。
- Destructive：`errorRed` 或 `errorRedSoft`，必须有确认。
- Disabled：浅灰背景 + `subtle` 文案，不参与核心对比要求。

### Inputs / Forms / Bottom Sheets

- 新增/编辑工时记录使用底部 sheet。
- Sheet 背景为 `surfaceRaised`，顶部可有短 drag handle。
- 表单字段白底或 `surface`，边框 `hairline`，圆角 14-18dp。
- 主按钮固定在表单底部或视觉末端。
- 删除当天记录、恢复备份、WebDAV 恢复等破坏性操作必须二次确认。

## 6. Layout Principles

- Mobile-first，优先 390dp 宽度。
- 页面左右边距 16dp。
- 卡片内边距 14-18dp。
- 卡片间距 10-14dp。
- Section 间距 18-24dp。
- 触控目标不小于 44dp，相邻触控目标至少 8dp 间隔。
- 可读性优先于信息密度；大字号下允许卡片变高，不能隐藏关键语义。

## 7. Dynamic Work Segment Rules

首页和日期详情必须兼容：

- 只有普通班；
- 普通班 + 一次加班；
- 普通班 + 多次加班；
- 多段普通班；
- 夜班跨天；
- 有补贴/扣款/备注；
- 当天无记录。

禁止实现：

- 固定 `普通 / 加班` 两格作为唯一摘要。
- 假设一天最多一次加班。
- 把多次加班挤进同一行导致换行破坏布局。
- 只用颜色区分普通、加班、夜班。

推荐实现：

- 顶部摘要使用可换行 chips。
- 下方使用 segment list 展示真实分段。
- 月视图只放摘要，文字细节交给列表和日期详情。

## 8. Page State Contracts

| 场景 | 空状态 | 成功状态 | 错误状态 | 部分完成 |
| --- | --- | --- | --- | --- |
| 首页今天无记录 | 说明“今天还没有记录”，主按钮补今天 | 保存后回到今日卡片和分段列表 | 保存失败保留 sheet，不吞掉输入 | 有分段但信息不完整时仍显示时间和净工时 |
| 日历当月无记录 | 月视图仍可选日期，列表视图给“新增第一段” | 新增后日期 cell、月汇总和日期详情同步 | 日期选择失败不改变当前选中日 | 月网格只显示摘要，文字细节交给列表和日期详情 |
| 汇总范围无记录 | 显示 0h、¥0 和“没有匹配日期”，仍可切范围 | 导出完成说明保存路径或取消 | 保存失败说明可重试或更换位置 | 自定义范围起止调整时立即刷新统计 |
| 备份/WebDAV 未配置 | 明确“未连接；可配置坚果云备份” | 保存配置后说明普通备份不含密码 | 缺密码、上传失败、限流、内容未变化要分开提示 | 自动备份开启但未授权时提示需重新授权 |
| 破坏性操作 | 不在默认主操作区出现 | 删除/恢复后刷新相关页面，并可从最近删除找回 | 取消不改变账本 | 删除单段和删除整天必须分开表达 |

## 9. Accessibility & Verification

至少覆盖：

- 首页、日历、汇总、设置在普通字号下主要内容不被底部导航遮挡。
- 日历在 2.0 text scale 下不抛布局异常。
- 日历日期 cell 不只靠颜色表达状态。
- 重要按钮有文字或 tooltip/semantics。
- 删除、恢复、覆盖类动作有确认弹窗。
- 正文文本对比至少 4.5:1；大号粗体文本至少 3:1。
- 触控目标不小于 44dp。

## 10. Explicit Non-Goals

- 不做企业后台式高密度表格。
- 不做复杂 BI 图表。
- 不做炫酷动效或品牌插画优先。
- 不做大面积暖黄、复古纸张、手账卡通风。
- 不为未来多端同步预埋复杂 UI。
- 不用设计系统抽象替代真实业务语义。
